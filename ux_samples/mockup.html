Dockerfile para app Next.js en fly.io

¿Me pasas un Dockerfile para desplegar una app Next.js 15 (output standalone) en fly.io? Región Madrid, VM pequeña, volumen montado en /data.

Aquí tienes una configuración mínima y funcional. Usa multi-stage build para que la imagen final sea lo más pequeña posible y aprovecha el output standalone de Next.js para no arrastrar node_modules en producción.

Dockerfile

dockerfile# Stage 1 — deps
FROM node:20-alpine AS deps
WORKDIR /app
COPY package.json package-lock.json ./
RUN npm ci

# Stage 2 — build
FROM node:20-alpine AS build
WORKDIR /app
COPY --from=deps /app/node_modules ./node_modules
COPY . .
RUN npm run build

# Stage 3 — runtime
FROM node:20-alpine AS runtime
WORKDIR /app
ENV NODE_ENV=production
COPY --from=build /app/.next/standalone ./
COPY --from=build /app/.next/static ./.next/static
COPY --from=build /app/public ./public
EXPOSE 3000
CMD ["node", "server.js"]

fly.toml mínimo

tomlapp = "claude-lite"
primary_region = "mad"

[build]

[[mounts]]
  source = "data"
  destination = "/data"

[http_service]
  internal_port = 3000
  force_https = true
  auto_stop_machines = true
  min_machines_running = 0

Pasos para desplegar

  1. Crea el volumen: fly volumes create data --region mad --size 1.
  2. Configura tus secretos: fly secrets set ANTHROPIC_API_KEY=….
  3. Lanza el primer deploy con fly deploy.

Recursos sugeridos

RecursoValorCoste aprox.
VMshared-cpu-1x 512MB~$3/mes
Volumen1GB en /data~$0.15/mes
TráficoPrimeros 100GB gratis$0

Con auto_stop_machines = true la máquina se duerme cuando no hay tráfico. El primer request después de dormir tarda ~2s en levantar, a cambio de coste cercano a cero en idle.

Dime si quieres que añada healthcheck, compresión Brotli o logging estructurado y lo meto en otra iteración.
